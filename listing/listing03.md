Что выведет программа? Объяснить вывод программы. Объяснить внутреннее устройство интерфейсов и их отличие от пустых интерфейсов.

```go
package main

import (
	"fmt"
	"os"
)

func Foo() error {
	var err *os.PathError = nil
	return err
}

func main() {
	err := Foo()
	fmt.Println(err)
	fmt.Println(err == nil)
}
```

Ответ:
```
<nil>
false

error - встроенный интерфейс, который определяет один метод Error(), возвращающий строку.
os.PathError - структура из пакета os, которая имеет метод Error(), возвращающий строку. Эта структура реализует интерфейс error
В функции Foo() мы объявили переменную err типа *os.PathError, которой присваем значение nil
Затем мы возвращаем err как error
В функции main() мы присваиваем переменной err возвращаемое значение типа error = nil
При выводе на печать мы получаем <nil>, так как у нас есть тип, но значение равно nil
При выводе на печать сравнение с nil, мы получаем false, так как это проверка на пустой интерфейс, коим err не является

Интерфейс это структура:
type iface struct {
	tab *itab
	data unsafe.Pointer
}
tab - указатель на структуру itab (Interface Table), которая хранит некоторые метаданные о типе и список методов, используемых для удовлетворения интерфейса:
data - указатель на фактическую переменную с конкретным (статическим) типом

type itab struct {
	inter *interfacetype // указатель на тип интерфейса
    _type *_type // указатель на тип данных, реализующих интерфейс
    ...
}

Пустой интерфейс - интерфейс, не имеющий определённых методов. Представляет из себя структуру:
type eface struct {
	_type *_type
	data  unsafe.Pointer
}

Если интерфейс не пустой, то в структуре есть *itab, в котором есть указатель на тип интерфеса и указатель , где есть поля с сигнатурами методов.
Если интерфейс пустой, то в структуре вместо *itab появляется *_type (указатель на itab заменяется указателем на тип)
```
